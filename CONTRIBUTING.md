# Contributing to Uzbekistan Economic Policy Engine

## Team Workflow

This project uses a **trunk-based development** workflow with short-lived feature branches and pull request reviews.

**Project Manager:** @nozimmjon
**Primary repo:** `CERR-Uzbekistan/Uzbekistan-Economic-policy-engine`

---

## Branching Convention

```
main                  # Production-ready, protected
feature/<name>        # New features or models     (e.g., feature/dsa-model)
fix/<name>            # Bug fixes                  (e.g., fix/io-decimal-bug)
i18n/<name>           # Translation work           (e.g., i18n/pe-model-uzbek)
docs/<name>           # Documentation changes      (e.g., docs/methodology-update)
```

**Rules:**
- Never push directly to `main` — always open a PR.
- Branch from `main`, keep branches short-lived (< 1 week).
- Delete your branch after merge.

---

## Commit Messages

Use clear, imperative commit messages. Prefix with the area of change:

```
Add QPM impulse response horizon selector
Fix IO model decimal locale bug for UZ locale
Update PE model translations (RU/UZ)
Rebuild FPP model from CAEM September 2025 data
```

**Avoid** giant commits that touch multiple models. One PR = one model or one feature area.

---

## Pull Request Process

1. Create a branch from `main` following the naming convention above.
2. Make your changes. Test locally by opening the HTML files in a browser.
3. Push your branch and open a PR using the PR template.
4. Request review from @nozimmjon (required for merge).
5. Address review comments, then the PM merges.

### PR Checklist (auto-included via template)
- Pages load without console errors
- Charts render correctly with sample data
- i18n keys work in all 3 languages (if applicable)
- Mobile layout is not broken (check at 375px width)

## Working on Models

Each model lives in its own folder and should remain **self-contained**:

```
model_name/
  index.html    # Complete page — HTML, CSS, JS inline
  *_data.js     # Auto-generated from R (DO NOT edit manually)
  *_data.json   # Raw data (DO NOT edit manually)
```

**Important:**
- Never edit `*_data.js` or `*_data.json` files directly — they are generated by R scripts.
- All user-visible text must support i18n (`data-i18n` attributes + translation keys).
- Use the shared design system: `../shared/design-system.css` variables and patterns.
- Test with all 3 languages before submitting a PR.

---

## Shared Modules

Files in `shared/` are used by all models. Changes here affect everything:

| Module | What it does |
|--------|-------------|
| `report-engine.js` | CSV/PNG/PDF export |
| `scenario-engine.js` | Save/load/compare scenarios |
| `ai-advisor.js` | AI policy analysis |
| `toast.js` | Toast notifications |

**If you modify a shared module, test all 6 models before submitting.**

## Local Development

No build tools needed. Just open any `index.html` in a browser:

```bash
# Quick local server (optional, for testing cross-origin)
python -m http.server 8000
# or
npx serve .
```

### Testing Checklist
- [ ] Open in Chrome and Firefox
- [ ] Check browser console for errors
- [ ] Test all 3 languages (EN/RU/UZ)
- [ ] Test at mobile width (375px) and desktop (1920px)
- [ ] Verify charts render with correct data
- [ ] Test export buttons (CSV/PNG/PDF)
