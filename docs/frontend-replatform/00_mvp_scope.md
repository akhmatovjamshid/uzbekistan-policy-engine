# Frontend Replatform MVP Scope

## In Scope (MVP v1)

Pages included in MVP:

1. Overview
2. Scenario Lab
3. Comparison
4. Model Explorer (basic)

Cross-cutting capabilities included in MVP:

- App shell and unified navigation
- Shared design system foundation
- Saved scenarios (create, tag, retrieve)
- Side-by-side scenario comparison (2-4 scenarios)
- Common output cards/charts/tables across pages
- Basic narrative generation block
- Explicit model attribution and caveats
- i18n foundation (EN/RU/UZ scaffolding)

## Out of Scope (Post-MVP)

- Full Knowledge Hub polish and deep content workflows
- Advanced collaboration workflows (comments/approval flows)
- Natural-language scenario builder
- Heavy automation/orchestration features
- Public-facing multi-tenant deployment complexity
- Advanced recommendation/ranking assistants

## Scope Discipline Rules

- A feature enters MVP only if it materially improves one of the top jobs-to-be-done.
- If a feature increases complexity without improving decision speed, trust, or comparison quality, defer it.
- Any addition that impacts architecture or timeline requires explicit MVP scope review.

## Primary MVP Jobs-to-be-Done

1. Understand current macro status quickly.
2. Create and run a policy scenario with clear assumptions.
3. Compare alternatives and trade-offs.
4. Export decision-ready outputs.
5. Verify model caveats and method basics when needed.
