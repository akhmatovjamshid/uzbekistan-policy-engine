# Session Log — 2026-04-16

## Goal
Audit the project for new tool opportunities, build a Python MCP server, then conduct a professional methodology audit of all 6 economic models.

## What was done

### 1. MCP server built and merged (PR #22)
Inspired by France's [datagouv-mcp](https://github.com/datagouv/datagouv-mcp), built a Python MCP server (`mcp_server/`) exposing all 6 economic models as 12 AI-queryable tools. Models ported from JavaScript to Python with numpy for matrix operations.

**12 MCP tools:** `list_models`, `qpm_impulse_response`, `qpm_baseline_forecast`, `cge_simulate`, `fpp_project`, `io_demand_shock`, `io_sector_info`, `pe_trade_impact`, `pe_trade_overview`, `dfm_nowcast`, `dfm_kalman_update`, `scenario_compare`

**Infrastructure:** Data conversion script (JS→JSON via Node.js), Dockerfile, unit tests for all 6 models (24 tests, all passing), `.mcp.json` for Claude Code auto-configuration.

**Dependencies installed:** numpy, scipy, mcp SDK, pydantic, uvicorn, python-dotenv.

### 2. REST API + Chat widget planned, then deferred
Designed a plan for REST API endpoints + AI chat widget for the web frontend. The MCP server already provides AI-powered analysis via Claude Code/Desktop.

### 3. Professional model methodology audit
Conducted a deep audit of all 6 models' equations, calibration, data, and methodology. Key findings:

**QPM:**
- b3 (external demand) parameter is dead code — no shock button
- Phillips Curve lacks direct import price pass-through (only indirect via RER gap)
- Baseline forecast (`runBL`) structurally inconsistent with IRF solver (ad-hoc `a2*rmc*3` scaling)
- No risk premium in UIP equation

**PE:**
- Uniform elasticity ε=1.27 for all 254K products — food and machinery treated identically
- Ad-hoc 0.6 regime filter has no economic justification

**FPP:**
- Inflation path aggressively optimistic (9.6%→3.0% in one year)
- Current account exogenous — no RER feedback
- Velocity treated as exogenous policy knob

**I-O:**
- Type I multipliers only — no induced consumption effects (underestimates by 20-40%)

**CGE:**
- No labor market — cannot assess employment effects
- Symmetric elasticities (σ_t = σ_q = 0.70) assumed, not estimated

**DFM:**
- Single factor may miss sectoral divergence
- Seasonal adjustment undocumented

**Cross-model:**
- QPM and FPP Phillips Curves have incompatible calibrations (a1=0.6 vs λ1=0.05)
- Only DFM has uncertainty bands
